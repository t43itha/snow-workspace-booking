[18, 19] Connecting your Personal Developer Instance (PDI) to GitHub lets you back up your custom applications, create a clean portfolio, and showcase your actual code to hiring managers. [1] ## Step 1: Create a Repository on GitHub [2] Before configuring ServiceNow, you need a blank repository to receive your code. 1. Log into your GitHub account. 2. Click New to create a repository. 3. Name your repository (e.g., servicenow-workspace-booking). 4. Set the visibility to Public (crucial for sharing with hiring managers). 5. Important: Leave "Add a README file", ".gitignore", and "Choose a license" unchecked. The repository must be completely empty. 6. Copy the Repository URL (e.g., https://github.com). [3, 4, 5, 6, 7] ------------------------------ ## Step 2: Generate a GitHub Personal Access Token (PAT) [8] ServiceNow cannot log into GitHub using your standard password. You must generate a secure token. [9, 10] 1. In GitHub, click your profile picture (top right) ➔ Settings. 2. Scroll down on the left menu and click Developer settings. 3. Select Personal access tokens ➔ Tokens (classic). 4. Click Generate new token ➔ Generate new token (classic). 5. Give it a clear note (e.g., ServiceNow PDI Connection). 6. Check the repo scope checkbox (this grants write access to your repositories). 7. Scroll to the bottom and click Generate token. 8. Copy the token immediately. You will never see it again once you close the page. [11, 12, 13, 14, 15] ------------------------------ ## Step 3: Create a Credential Record in ServiceNow Now, you need to store your GitHub token securely inside your PDI. 1. Log into your ServiceNow PDI. 2. In the Application Navigator, search for Connections & Credentials ➔ Credentials. 3. Click New. 4. Select Basic Auth Credentials. 5. Fill out the form fields exactly as follows: * Name: GitHub Credentials * User name: Your GitHub username (case-sensitive) * Password: Paste the Personal Access Token (PAT) you copied in Step 2. [16, 17, 18] 6. Click Submit. ------------------------------ ## Step 4: Link Your Scoped App to GitHub With the security clearance in place, you can now link your custom application. 1. In ServiceNow, navigate to System Applications ➔ Studio. 2. Select the Scoped Application you want to link. 3. Look at the top left menu of Studio and click Source Control ➔ Link to Source Control. [19] 4. Complete the configuration form: * URL: Paste your GitHub Repository URL from Step 1. * Credential: Click the magnifying glass and select your GitHub Credentials record. * Branch: Leave this as master or main. * Commit comment: Type a starting message like Initial PDI backup. [20] 5. Click Link to Source Control. [21] Once the progress bar finishes loading, your entire application structure, configurations, and JavaScript files will be pushed to your GitHub repository automatically. ------------------------------ ## 🔄 How to Push New Updates As you continue coding and developing your app in Studio, your changes will not update on GitHub automatically. Whenever you hit a milestone, click Source Control ➔ Commit Changes inside Studio to push your latest progress to your public portfolio repository.
